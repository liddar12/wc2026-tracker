# RJ30.2 — Match Intelligence (all $0; AI narrative via existing dormant pipeline)

Free, research-backed increment. Data source = **ESPN summary endpoint** (already used by the app),
which exposes per-team `possessionPct, totalShots, shotsOnTarget, blockedShots, passPct,
accuratePasses, saves, tackles, fouls, offsides, crosses` + a `keyEvents` timeline — verified live.

## Stories + acceptance criteria
- **As a fan**, I want a **Match Stats** panel on the matchup page (possession bar, shots/on-target, passing %, key stats) so I can see how a game is unfolding.
  - GWT: given a match with ESPN boxscore stats in `data/match_stats.json`, when the matchup renders, then a stats panel shows possession (a two-sided bar summing ~100), shots & on-target, passing %, and 4-6 key stats per team; absent stats → graceful empty/omit (no zeros-for-everything).
- **As a fan**, I want a **momentum chart** so I can see the flow (pressure/goals over time).
  - GWT: given `keyEvents`, when rendered, then a compact timeline/sparkline marks goals + shot pressure by minute; reduced-motion honored; no data → omit.
- **As a fan**, I want a **shots vs xG** read so I can judge finishing.
  - GWT: show ESPN real shots/on-target next to the app's MODEL xG (`xg.json`) for the pair; label clearly as model xG (not event xG).
- **As a fan**, I want **smart insights** (free, computed) — e.g., "Dominant possession", "Clinical: 2 goals on 4 shots on target", "Model favored X; shots agree/disagree".
  - GWT: `app/lib/match-insights.js` derives 1-3 plain-language insight lines from stats deterministically; $0, always on; safe when stats missing.
- **As a fan (with the AI key set)**, I want an **AI match analysis** that reads the real stats.
  - GWT: `generate_previews.py` recap prompt includes `match_stats`; dormant with no `ANTHROPIC_API_KEY` (no UI change); active → 1-2 sentence stats-aware recap.
- **As an iPhone user**, I want a **home-screen widget** + **Siri** despite PWA limits.
  - GWT: ship `widget/scriptable-wc26.js` (free Scriptable app) that fetches the site's `/data/*.json` and renders next-match/live-score on the Home Screen; `docs/SIRI_AND_WIDGET.md` with install steps + a Siri Shortcut recipe (opens the site / next match).

## QA (node:test + Playwright)
- `match-insights` unit: dominance/clinical/model-agreement lines from fixtures; empty-stats safe.
- `match-stats` render: possession bar sums ~100; shots/passing present; empty-state when absent; scrollWidth≤390.
- `momentum-chart`: builds from keyEvents; reduced-motion; empty → empty fragment.
- `scrape_match_stats --self-test`; `generate_previews` stats-aware, dormant no-key path unchanged.
- `scriptable-wc26.js` `node --check` + a pure-logic test of its data-shaping.
- Regression: full 4-step gate 100% green; no matchup-detail regressions.

## Build partition
### Wave 1 (parallel, NEW files only — no shared files)
- **A (pipeline+insights):** `scripts/scrape_match_stats.py`* → `data/match_stats.json`* (per match_id: teamA/teamB stats + keyEvents); `app/lib/match-insights.js`* (free computed insights); tests.
- **B (components):** `app/components/match-stats.js`* (panel: possession/shots/passing + insights + shots-vs-model-xG); `app/components/momentum-chart.js`* (reuse sparkline); tests.
- **C (iOS + AI):** `widget/scriptable-wc26.js`*, `docs/SIRI_AND_WIDGET.md`*, extend `scripts/generate_previews.py` (stats-aware recap, dormant); tests.

### Wave 2 (integrator, shared files)
`app/views/matchup-detail.js` (mount match-stats + momentum) · `app/data-loader.js` (load `match_stats.json` → `data.matchStats`) · `app/styles.css` (stats/momentum CSS) · `scripts/validate_data.py` (match_stats coverage, warn-only) · `.github/workflows/{frequent,live}_update.yml` (`scrape_match_stats.py` step) · `app/lib/strings.es.js` + i18n keys for stat labels.

## iOS / a11y
390px (stats bars use % widths, tabular-nums, scrollWidth≤390); reduced-motion on momentum; aria-labels on the possession/stat bars; reuse existing `.home-card`/tokens; no new design language. Widget script is user-installed (free Scriptable), reads public JSON only.

## Deploy
Auto-deploy on 100% green (race-safe). AI analysis stays dormant until `ANTHROPIC_API_KEY` is set (same handoff as RJ30.1 previews).
