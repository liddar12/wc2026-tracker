# WC2026 Tracker — iOS Design Overhaul (Audit + P0/P1/P2 Backlog)

**Date:** 2026-07-01 · **Method:** 22-agent parallel audit (16 per-page iOS-design critics + 6 specialists: color, type/space, motion, navigation/IA, circular-bracket, a11y/dark-mode) over screenshots of every public page captured at iPhone 390×844 (light + key dark). **Findings:** 40 P0 · 88 P1 · 85 P2.

Per owner decision this ships as **two selectable release tracks** — **Track A: Polish + Signature Moments** (refine the current look on the existing vanilla-JS architecture, incrementally) and **Track B: Bold Redesign** (a new iOS-native visual language). Nothing is built until the owner picks. The circular/radial bracket is mocked first (see `/tmp/wc-audit/circular-bracket-{polish,bold}.png`) and, if approved, ships as an additive toggle.

## 2. The five recurring problems (why the app reads "web dashboard," not "iPhone app")

1. **Shouty ALL-CAPS typography** (18 findings) — every header is caps + tracking ("TODAY'S MATCHES", "WHEN & WHERE & HOW TO WATCH"). iOS uses sentence-case grouped-list headers + a per-screen Large Title. Flattens hierarchy, reads dated.
2. **Top scrollable tab bar** (12 findings, many P0) — Home/Schedule/Projected/Venues/› with destinations hidden off-screen; the classic iOS anti-pattern. iOS wants a fixed **bottom** tab bar.
3. **Incoherent color** (15) — orphan teal chrome bar that matches nothing; two token systems (dead navy/red under live teal/coral); a confirmed **dark-mode contrast failure** (black date text on cyan hero, ~2:1); no gold/tournament warmth; "brand accent" and "danger red" collide on one hue.
4. **Truncation & broken states** (P0s) — `ENGLA…` / `DR CO…` on the most important card; raw `round_of_32`; empty Position-Ratings bars; blank flag placeholders; empty "TODAY'S MATCHES" header card.
5. **Data-state correctness** (P0s) — a **finished** match still shows pre-match "50% / 50%" + "HYBRID PICK Japan" (Japan lost); a **live** match shows no liveness. The UI doesn't respect match state.

## 3. Track A — Polish + Signature Moments

Refines the current identity on the existing architecture. High impact, low risk, ships in slices. Mostly `polish`/`both`.

### P0 — fix what looks broken (touches every page)
- **A1 · Typography reset.** Replace ALL-CAPS letter-spaced headers with iOS grouped-list headers (sentence case, 13pt semibold, secondary color); add one Large Title per screen. Adopt the type scale in §6.
- **A2 · Team-name truncation.** Never clip proper nouns. Flag + 3-letter FIFA code (ENG, COD) as the anchor; full names wrap at Dynamic Type sizes.
- **A3 · Raw enum leaks.** `round_of_32` → "Round of 32" everywhere (stage pills, venue detail).
- **A4 · Decided-match correctness.** On FT games, stop rendering pre-match 50/50 + "HYBRID PICK <loser>" + "YOUR PICK" as if live. Collapse to a retrospective "Model called it ✓ / missed it ✗" and the final result.
- **A5 · Live-match state.** In-progress matchup gets a red LIVE badge + pulsing dot + minute; live win-prob labeled as live.
- **A6 · Dark-hero contrast.** Darken the hero gradient in dark mode (`#0E5561→#0A3F49`) and make on-hero text adaptive white — kill black-on-cyan (WCAG AA).
- **A7 · Empty/placeholder states.** Real values or honest empty copy for Position Ratings, unresolved-slot flags (W79/W80), and the empty "Today's matches" card.

### P1 — notable polish
- **A8 · Color unification** (Polish palette §5A): delete dead navy/red tokens; add a `--gold` champion accent; split `--danger` (team-B/upset) from `--accent` (brand/CTA).
- **A9 · Match-card consistency:** one banner rule — split home/away flag gradient with a legibility scrim, in every state; differentiate status via pill/score, not banner geometry; fix winner-name contrast.
- **A10 · Spacing rhythm:** 8pt grid; section headers as plain text (not their own card); remove empty header-card containers; 16pt card padding.
- **A11 · Tame the giant pages:** sticky segmented sub-nav + section anchors + back-to-top for Golden Boot (11k) and Model Accuracy (10k); unify back-nav (kill golden-boot's ad-hoc "← Home").
- **A12 · Motion P0→ kill the `innerHTML` full-flash rebuild** (diff/crossfade + View Transitions API, preserve scroll/underline/what-if state — directly retires the re-render flash the perf work exposed); add pressable press-scale + haptics; skeleton→content fade. Motion tokens in §7.
- **A13 · Iconography:** standardize on SF Symbols (one weight, label-tint); drop pastel tiles + decorative emoji in headers.
- **A14 · Accessibility:** contrast tokens (§8), Dynamic Type, VoiceOver grouping (a score reads as one element), 44pt tap targets.

### P2 — nice-to-have
- **A15** native pull-to-refresh (replaces the passive "24m ago"). **A16** tabular figures for all numeric columns. **A17** compact match-card variant + segmented Today/Upcoming/Results on Home. **A18** bracket re-cascade wave, badge count-ups.

### ★ Signature moment
- **A19 · Circular/radial bracket** as an additive `layout=radial` toggle on Projected (§9). **Reuses `buildAutofill(data,source,{overrides})` + `OVERRIDES` + `confidence()` + `flagFor()` — zero model/data changes**, same tap→override→paint loop. Mockups rendered (polish + bold).

## 4. Track B — Bold Redesign

A new iOS-native visual language. Bigger change, higher "wow," more regression surface. Mostly `bold`.

- **B1 · P0 · Bottom tab bar** (5: Home · Schedule · Bracket · Play · More) with SF Symbols + haptic selection; segmented sub-navs (Bracket = Projected\|Live\|Mine; Play = Pools\|My Picks\|Groups); long-tail → More hub. Retires the top scroll-tabs.
- **B2 · P0 · iOS Large Titles** per screen (collapse-on-scroll); retire the orphan teal wordmark bar; header shares the content surface.
- **B3 · P0 · Bold tournament palette** (§5B): dark tournament canvas + gold champion energy — the World-Cup-grade identity.
- **B4 · P1 · Inset-grouped list system** + full component redesign (richer match cards, elevation ramp).
- **B5 · P1 · Circular bracket as the hero** Projected view (dark/gold, optionally default).
- **B6 · P1 · Shared-element flag transition** into the matchup detail.
- **B7 · P1 · Search** (teams / matches / players).
- **B8 · P2** Home segmented Today/Upcoming/Results + compact cards (3–4 above fold). **B9 · P2** live-activity-style live-match surface.


## 8. Accessibility / dark-mode fixes
- Hero on-gradient text adaptive (white dark / `#0B1220` light) — no black-on-cyan.
- Muted text → `#B7C0CE` on dark (~8.7:1); eyebrow/caption `#9AA4B2` (~5.2:1).
- Confidence pills: `#06210F` on `#4ADE80` (~8:1).
- Dark elevation ramp: `#0F1115 · #171A21 · #1E222B · #262B36`, hairline `#2E3340`.
- VoiceOver: group each match to one element ("England 2, DR Congo 1. Final, Round of 32. Mercedes-Benz Stadium."). Star = button w/ label+value. 44pt hit targets on all pills/chips/★/ⓘ. Dynamic Type resilient (wrap, don't clip).

## 9. Circular / radial bracket (mock-first signature)

**Feasibility:** pure alternate projection of existing data — `buildAutofill` returns flat `[{matchNumber,team,team_a,team_b}]` for matches 73–104, `OVERRIDES{matchNumber:team}` drives re-cascade, `confidence()`→0–100, `flagFor()`→emoji. Ships as `layout=radial` on the existing Projected route reusing the same tap→override→paint loop. Vanilla SVG (`viewBox 0 0 390 ~460`, ~63 wedge paths + 32 flag nodes). No framework, no perf concern.

**Ring map (center→out):** Champion (center disc) · Final (2) · SF (4) · QF (8) · R16 (16) · R32 leaves (32, outer rim @11.25° pitch). Match #: R32 73–88 · R16 89–96 · QF 97–100 · SF 101–102 · Final 104 · 3rd 103 as a chip below the wheel.

**Legibility rule:** flags always on rim + winning path; **names/confidence appear only for the focused wedge (tap-to-expand) or the champion**. Bold-dark radial with a glowing gold champion core is the lead recommendation.

**Mockups:** `/tmp/wc-audit/circular-bracket-polish.png` (light/teal), `/tmp/wc-audit/circular-bracket-bold.png` (dark/gold).

## 10. Recommendation & sequencing

1. **Track A P0 first** (A1–A7): typography + truncation + decided/live correctness + dark contrast + empty states. Highest impact, lowest risk, touches every page, ships in days. Each item is a scoped change through the existing regression gate + auto-deploy.
2. **Add the circular bracket (A19)** as the signature moment — cheap (reuses plumbing), high delight, low risk (additive toggle).
3. **Track A P1** (color unify, card consistency, spacing, giant-page nav, motion-flash kill, icons, a11y).
4. **Then decide on Track B** (bottom tab bar + large titles + bold palette) as a deliberate identity shift — the biggest "feels native" jump, but the largest regression surface, so it warrants its own gated phase.

## Appendix A — per-page critique
#### Home
_It reads as a competent web dashboard skinned for mobile, not a native iOS app: lots of shouty micro-caps section labels, floating rounded cards on a gray field, and truncated team names in the hero result. There's a real spark in the flag-gradient match cards, but the hierarchy is flat and the chrome (top scrollable tabs, generic gear/person icons) telegraphs "PWA" rather than "iPhone app."_

- **P0** (both, typography): Team names are truncated in the hero result card: 'ENGLA...' and 'DR CO...' The single most important match on the page can't spell out who is playing. This looks broken and undermines trust in the data. → _Use FIFA 3-letter codes (ENG, COD) beside the flag, or drop to a two-line/smaller SF weight that fits full names. Reserve the flag + score as the visual anchor and let names wrap rather than clip. Never truncate a proper noun on the primary card._
- **P0** (polish, typography): Nearly every label is ALL-CAPS with heavy letter-spacing: TODAY'S MATCHES, FULL SCHEDULE, RECENT RESULTS, JUMP TO, LIVE ELO MOVERS, FIFA WORLD CUP 2026, FINAL, TODAY, R32. This is the opposite of the iOS convention (sentence case, SF weights). It reads as loud, dated, and web-templated, and it flattens hierarchy because everything shouts equally. → _Adopt iOS grouped-list header style: sentence case, 13pt semibold, secondaryLabel color, tighter tracking (e.g. 'Today's matches', 'Recent results'). Keep caps only for true chips like status pills, and even there prefer 'Final' over 'FINAL'._
- **P1** (bold, iOS-nativeness): Navigation is a scrollable TOP tab bar (Home / Schedule / Projected / Venues / >). Horizontal top tabs with a peeking chevron are an Android/web pattern; they're not thumb-reachable and hide destinations off-screen. There's also no bottom tab bar, which is the core iOS navigation affordance. → _Move primary destinations to a fixed bottom UITabBar (5 max: Home, Schedule, Bracket/Projected, Pools, More) with SF Symbols and haptic selection. Demote overflow to a 'More' tab. This frees the top for a large-title header and makes the app feel native instantly._
- **P1** (both, IA): The header wastes the most valuable real estate: a generic soccer-ball logo + 'World Cup 2026 App' wordmark + gear + Sign in, on a teal bar that doesn't match any other surface color. 'App' in the title is redundant and amateurish. → _Replace with an iOS large-title ('World Cup 2026' as a real large title that collapses on scroll), drop the '...App' suffix, and move Sign in into a Profile/More tab or a top-right avatar. Retire the standalone teal bar so the header shares the app's surface palette._
- **P1** (polish, color): The teal header is an orphan color — it appears nowhere else in the UI and clashes with the muted blue-gray date hero and the red/green flag cards. There's no coherent accent system; the pink 'ENGLA...' text and pink tab underline are yet another unrelated hue. → _Define one tokenized accent (pick one: FIFA-blue or a considered green) used for the tab indicator, links, and interactive text. Make the header transparent/blurred over the content surface so the palette is driven by content (flags), not chrome._
- **P1** (polish, spacing): Rhythm is loose and inconsistent: the 'TODAY'S MATCHES' card is an empty white pill with just a label and a large void beneath it, match cards have generous but uneven internal padding, and gaps between cards don't sit on a consistent 8pt step. The empty label-only card reads like a loading/broken state. → _Make section headers plain text on the background (not their own card), and let match cards sit directly under them at a consistent 12–16pt gap. Standardize card internal padding to 16pt and vertical rhythm to multiples of 8. Remove the empty header-card container entirely._
- **P1** (both, component): Match cards are inconsistent between states. The finished match shows a diagonal blue gradient wedge (only top-right), while upcoming matches show full red/green flag gradients across the whole banner. Same component, three different fills — the logic isn't legible and looks accidental. → _Define one banner rule: split gradient = home flag on left, away flag on right, meeting at center, for every card regardless of state. Differentiate status with the pill/score, not by changing the banner geometry._
- **P1** (polish, iconography): Mixed icon languages: the JUMP TO grid uses flat pastel-tinted icons in soft squares (a webby, Material-ish look), while emoji appear elsewhere (⚠️/fire on 'DON'T MISS', trophy). This blend of pastel glyphs + emoji + line icons has no single visual voice. → _Standardize on SF Symbols throughout (one weight, label-color tint on the app surface — drop the pastel tinted tiles). If you want color, tint the symbol itself, not a background chip. Remove decorative emoji from section headers._
- **P2** (polish, empty-state): The 'TODAY'S MATCHES' card renders as a header with no visible content in the fold, which is ambiguous — is it loading, empty, or just a label? There's no skeleton or explicit empty copy. → _If matches exist, pull the first card up under the header immediately. For a true empty day, show an illustrated/system empty state ('No matches today — next kickoff Fri 3:00 PM') rather than a blank pill._
- **P2** (bold, information-density): Density is low for a data app: each match card consumes a huge vertical band for one line of info (teams, score, venue, status). A fan scanning 'today' has to scroll a lot to see three games, and the 'ALL 104 MATCHES →' and results table feel far down. → _Offer a slightly more compact card variant (venue on the same row as status, tighter banner height ~72pt) so 3–4 matches fit above the fold. Consider a segmented control (Today / Upcoming / Results) instead of stacking every section._
- **P2** (polish, a11y): Pink team-name text ('ENGLA...') on white and the light-gray metadata ('Mercedes-Benz Stadium', 'R32' chips) look low-contrast, and the caps+tracking hurts legibility. No evidence of Dynamic Type support given the fixed truncation. → _Verify 4.5:1 contrast on all text over banners and cards; darken secondary metadata to iOS secondaryLabel. Support Dynamic Type so names scale instead of truncating, and test at larger accessibility sizes._
- **P2** (bold, layout): Safe-area / large-title patterns aren't used: content starts right under a hard-edged colored bar with no large title, and cards float on a gray backdrop giving a boxy, web-card feel rather than iOS inset-grouped lists. → _Adopt inset-grouped list styling (systemGroupedBackground) or continuous full-width cards with respected safe-area insets, plus a collapsing large title. This alone shifts the feel from 'website' to 'app.'_
- 💡 **Signature:** The split flag-gradient match card is the app's soul — lean into it: make every card a true left-home / right-away flag gradient meeting at the score, and when a live match updates, animate the winning side's color subtly blooming across the banner with a soft haptic tick. A 'today' card that literally wears both nations' colors and pulses on a goal is a moment no other bracket app owns.

#### Schedule
_Legible and information-dense, but it reads like a responsive web page squeezed into a phone, not a native iOS app: shouty condensed all-caps team names that truncate mid-word, a top scroll-tab bar instead of a bottom tab bar, and a busy per-card gradient header that fights the content underneath._

- **P0** (both, typography): Team names are truncated mid-word and embarrassingly: 'ENGLA...' for England and 'DR CO...' for DR Congo in the top card. A schedule app whose one job is telling you who is playing cannot hide the team name. The condensed all-caps face makes it worse because the ellipsis lands mid-word. → _Give team names room: drop the flag+name+score to a two-line layout if needed, or reserve fixed columns with the score centered. Use full country names (or official 3-letter FIFA codes ENG / COD as a deliberate fallback), never a raw CSS ellipsis. Switch from the condensed ALL-CAPS face to SF Pro sentence-case so names fit and wrap gracefully._
- **P1** (polish, typography): Everything shouts. Team names, FINAL, TODAY, R32, PARLAY OF THE DAY, section labels are all ALL-CAPS with heavy letter-spacing. That is a sports-broadcast/web convention, not iOS. It flattens hierarchy (the team name and a 'FT' chip compete) and reduces the legibility that drives the truncation. → _Adopt the iOS type ramp: sentence-case team names in SF Pro Semibold ~17pt, secondary metadata in Regular ~13pt secondaryLabel gray, and reserve uppercase only for tiny caption chips (R32) at ~11pt. Let weight and color carry hierarchy, not caps + tracking._
- **P1** (bold, iOS-nativeness): Primary navigation is a horizontally scrollable TOP tab bar (Home, Schedule, Projected, Venues, chevron-more). On iPhone this is the least reachable spot on the screen, hides destinations behind a scroll, and is a web pattern. It also means the app has no persistent bottom chrome, so momentum scrolling runs edge to edge with no anchor. → _Move primary sections to a native bottom UITabBar (5 items max: Schedule, Projected, Venues, More) with SF Symbols + labels, respecting the home-indicator safe area. Keep the day strip as an in-page control under a large title. Add a haptic tap on tab/day change._
- **P1** (bold, color): Each match card carries a full-bleed diagonal gradient header derived from team colors (England blue, Belgium red/green, USA blue) plus a faint watermark ball. Stacked down the feed it's visually noisy, the gradients have no functional meaning, and the ghosted watermark reads as a rendering artifact. It also crowds the flags sitting just below it. → _Replace the gradient band with a quiet card: system background, a thin hairline separator, and let the two flags + a subtle winner accent do the color work. If you want team identity, use a slim 3-4pt colored top rule instead of a 25% tall gradient. Drop the watermark ball entirely._
- **P1** (polish, layout): Card internal rhythm is off the 8pt grid: the gradient header, flag row, and metadata row have inconsistent vertical padding, and the flags float with uneven optical spacing to the team names. The score/kickoff center column isn't baseline-aligned with the flanking team names, so BELGIUM / 3:00 PM / SENEGAL sit at slightly different heights. → _Rebuild the card on an 8pt vertical rhythm (16pt card padding, 8pt between rows). Use a 3-column grid (flag+name | score/time | name+flag) with a shared baseline and center the middle column. Standardize flag size (e.g. 28x20) and gap._
- **P2** (both, iconography): Component inconsistency between states: the FINAL card shows a big black score, upcoming cards show a kickoff time in the same slot but in the condensed caps face, and there's no live/upcoming/final visual system beyond the tiny FT/TODAY chips. A user can't pre-attentively tell a finished match from an upcoming one. → _Introduce a consistent status system: colored dot or SF Symbol per state (gray checkmark = final, blue dot = upcoming/today, pulsing red = live), placed consistently. Keep the score slot for scores and the time slot for times, styled identically across cards._
- **P2** (polish, IA): The PARLAY OF THE DAY card sits below the first few match cards rather than being a clearly-owned hero, and it visually resembles a match card enough to blur the two content types. Its own internal type is also very dense (three tiers x four bets, all-caps labels, model-est badges). → _Give the parlay card a distinct treatment (accent-tinted background or a labeled section header 'Today's parlay') so it reads as an editorial/betting module, not another fixture. Add generous 8pt spacing between the three tiers and use sentence case for bet labels._
- **P2** (both, a11y): No visible support for Dynamic Type or large-title patterns, and the condensed all-caps truncation will get dramatically worse at larger text sizes. The 'Wednesday, July 1, 2026' date acts as a heading but isn't presented as an iOS large title, and there's no visible momentum/large-title collapse behavior. → _Use a real iOS large-title nav bar for the day ('Wednesday, Jul 1') that collapses on scroll. Make all text scale with Dynamic Type and let team-name rows grow to two lines instead of truncating. Verify contrast of the pink team-name text on the light card._
- **P2** (polish, color): The pink/red team-name color (ENGLA...) is decorative and inconsistent — it's unclear whether red means 'home', 'winner', or is just brand styling, and it reduces contrast versus plain label black. → _Reserve color for meaning: tint only the winning team's name (or none), keep the rest in primary label color. Pick a single semantic accent and apply it consistently across the app._
- 💡 **Signature:** When a match goes final, animate the result inline: the two flags slide a hair toward center, the winner's name settles into a bold semantic accent while the loser dims to secondary, the score counts up with a spring, and a crisp success haptic fires — a single tactile beat that makes 'FINAL' feel earned instead of a gray chip.

#### Projected bracket tree
_It reads like a competent data table pretending to be a bracket — the content is genuinely interesting, but the chrome (teal top bar, floating chips, gray boilerplate) feels like a generic web dashboard, not a native iOS surface you'd want to swipe through._

- **P1** (bold, iOS-nativeness): Primary navigation (Home / Schedule / Projected / Venues / >) is a scrollable TOP tab row with an off-screen chevron. This is an Android/web pattern; it competes with the model chips and stage nav directly below it, creating three stacked horizontal scrollers the user must mentally disambiguate. → _Move primary sections to a native bottom UITabBar (SF Symbols, 4-5 items, safe-area inset). Reserve the top for a large-title 'Projected' header that collapses on scroll. This frees the top and kills the hidden-chevron discoverability problem._
- **P1** (both, typography): 'MODEL', 'ROUND OF 32', 'ROUND OF 16' are ALL-CAPS with letter-spacing — the shouty web-eyebrow look. iOS uses SF weights and sentence/Title Case for section headers (see Settings, Health). It fights the SF-native feel. → _Switch section labels to SF sentence case with weight for hierarchy: 'Model' as a footnote-gray label, 'Round of 32' as a semibold subheadline. Drop the tracking._
- **P1** (both, spacing): Vertical rhythm is off the 8pt grid — match cards, the R16 column, and the R32 column don't align to a shared baseline, so R16 cards float at arbitrary Y offsets relative to their R32 feeders. The eye can't trace which two teams feed a given R16 slot. → _Snap all card heights and gaps to an 8pt module and vertically center each R16 card on the midpoint of its two feeder R32 cards (classic bracket geometry). Add faint connector lines so advancement is literally traceable._
- **P1** (polish, color): The teal/petrol top bar is a different color family from everything below it (cool grays, green pills, blue winner text). #### Settings
_It reads as a competent responsive web settings form squeezed into a phone, not an iOS Settings experience. The bones are sensible (grouped cards, clear labels), but the ALL-CAPS letter-spaced headers, the top scrollable tab bar, and the flat red/teal web buttons instantly break the native illusion._

- **P0** (bold, IA): Primary navigation is a horizontally scrollable TOP tab bar (Home, Schedule, Projected, Venues, chevron>). Free the top for a large-title 'Settings' header._
- **P1** (both, typography): Every section header is ALL-CAPS with heavy letter-spacing (FAVORITE TEAM, LANGUAGE, MATCH ALERTS, THEME, MOTION, ACCOUNT, MODEL & ANALYTICS). - **P1** (both, layout): No large-title pattern and no clear screen title.
